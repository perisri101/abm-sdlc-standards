New Section 8 — Data Security & Integration Standards
1 New §8: Data Security & Integration Standards Inserted after §7 Development Standards

A full new section was inserted after Development Standards, triggering renumbering of §8–§18 → §9–§19. The section contains five subsections:

  • 8.1 Data Classification — Restricted / Confidential / Internal / Public with encryption and masking requirements
  • 8.2 Encryption Standards — TLS 1.2+ in transit, AES-256 at rest, OAuth 2.0, SFTP/FTPS for file transfers, Key Vault for secrets
  • 8.3 Hub-and-Spoke Integration Architecture — Boomi as central hub, ADR required for any direct P2P bypass
  • 8.4 Integration Tooling Matrix — Boomi, Azure Logic Apps, Audatex/Claims, Oracle OIC, Power Automate with data tier and governance owner
  • 8.5 PII Identification & Handling — 6-step PII assessment process; mandatory InfoSec data flow inventory registration for external PII flows

ISO tags: A.8.11 (Data masking), A.8.26 (Application security requirements). ISO mapping table updated accordingly.
